# AUR packaging

`octra-wallet` is packaged for Arch Linux / AUR from upstream
[`octra-labs/webcli`](https://github.com/octra-labs/webcli) commits.
There are no versioned upstream tags, so each AUR release is pinned to a
specific commit SHA.

## Files

- `PKGBUILD` — build recipe. `_commit` pins the upstream commit, `pkgver`
  is `<YYYYMMDD>.<short-sha>` (sortable, one release per commit).
- `.SRCINFO` — regenerated by the release workflow.
- `octra-wallet.sh` — `/usr/bin/octra-wallet` launcher. Creates
  `$XDG_DATA_HOME/octra-wallet` (default `~/.local/share/octra-wallet`),
  symlinks read-only assets from `/usr/share`, runs the binary from
  there so `data/wallet.oct` stays per-user.

## Install

```
yay -S octra-wallet
```

Run `octra-wallet` and open http://127.0.0.1:8420.

## Cutting a release

Trigger the **release** workflow manually in GitHub Actions:

- Leave `commit` blank → packages the current `HEAD` of
  `octra-labs/webcli:main`.
- Set `commit` to a specific upstream SHA → packages that exact commit.

The workflow:

1. Resolves the commit on `octra-labs/webcli` and pulls its author date
   via the GitHub API, derives `pkgver=YYYYMMDD.shortsha`.
2. Rewrites `_commit` and `pkgver` in `PKGBUILD`.
3. Fetches `https://github.com/octra-labs/webcli/archive/<sha>.tar.gz`
   and computes the real `sha256sums`.
4. Runs `makepkg -s` inside `archlinux:latest`.
5. Regenerates `.SRCINFO` with `makepkg --printsrcinfo`.
6. Creates a GitHub release tagged `pkg-<pkgver>` with the
   `.pkg.tar.zst` attached.
7. Pushes the updated `PKGBUILD` + `.SRCINFO` + wrapper to AUR over SSH.

## Required GitHub secrets

| Secret                 | Value                                                           |
| ---------------------- | --------------------------------------------------------------- |
| `AUR_USERNAME`         | Your AUR account name                                           |
| `AUR_EMAIL`            | Email registered on AUR                                         |
| `AUR_SSH_PRIVATE_KEY`  | Private key (full PEM) whose public half is on your AUR profile |

Generate with `ssh-keygen -t ed25519 -C aur -f aur_key`, paste
`aur_key.pub` into https://aur.archlinux.org/account, put the private
`aur_key` into the `AUR_SSH_PRIVATE_KEY` secret.

## Initial AUR submission

The workflow updates an existing AUR package; it does not create one.
Do the first submission manually:

```
git clone ssh://aur@aur.archlinux.org/octra-wallet.git aur-octra-wallet
cp PKGBUILD .SRCINFO octra-wallet.sh aur-octra-wallet/
cd aur-octra-wallet
git add .
git commit -m "initial import"
git push
```

Subsequent updates come from the release workflow.
