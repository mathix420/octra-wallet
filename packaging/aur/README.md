# AUR packaging

`octra-wallet` is packaged for Arch Linux / AUR. Sources in this directory:

- `PKGBUILD` — build recipe (source package; compiled on the user's machine)
- `.SRCINFO` — generated metadata; regenerated by the release workflow
- `octra-wallet.sh` — `/usr/bin/octra-wallet` wrapper. Creates
  `$XDG_DATA_HOME/octra-wallet` (default `~/.local/share/octra-wallet`),
  symlinks the read-only `static/` assets from `/usr/share`, and runs the
  binary from there so `data/wallet.oct` stays per-user.

## Install (end users)

```
yay -S octra-wallet
# or
paru -S octra-wallet
```

Then run `octra-wallet` and open http://127.0.0.1:8420.

## Local test build

From the repo root:

```
cd packaging/aur
# build from a published tag (requires the tag to exist upstream)
makepkg -si
```

To test an unreleased commit, build the binary directly with `make` from
the repo root — the PKGBUILD is tag-driven.

## Release flow

Pushing a tag of the form `vX.Y.Z[-suffix]` triggers
`.github/workflows/release.yml`:

1. Rewrites `pkgver` / `_gittag` to match the tag.
2. Computes `sha256sums` for the upstream tarball and wrapper script.
3. Runs `makepkg -s` inside an `archlinux:latest` container.
4. Regenerates `.SRCINFO` via `makepkg --printsrcinfo`.
5. Uploads the built `.pkg.tar.zst` to the GitHub release.
6. Pushes the updated `PKGBUILD` + `.SRCINFO` to the AUR over SSH.

`pkgver` cannot contain `-`, so `v0.04.10-alpha` becomes `0.04.10.alpha`.

## Required GitHub secrets

The AUR push step needs three repository secrets:

| Secret                 | Value                                                    |
| ---------------------- | -------------------------------------------------------- |
| `AUR_USERNAME`         | Your AUR account name                                    |
| `AUR_EMAIL`            | Email registered on AUR                                  |
| `AUR_SSH_PRIVATE_KEY`  | Private key (full PEM) whose public half is on your AUR profile |

Generate with `ssh-keygen -t ed25519 -C aur -f aur_key`, paste
`aur_key.pub` into https://aur.archlinux.org/account (SSH Public Key),
and put the private `aur_key` into the `AUR_SSH_PRIVATE_KEY` secret.

## Initial AUR submission

The workflow assumes the package already exists on AUR. For the first
submission do it once by hand:

```
git clone ssh://aur@aur.archlinux.org/octra-wallet.git aur-octra-wallet
cp PKGBUILD .SRCINFO octra-wallet.sh aur-octra-wallet/
cd aur-octra-wallet
git add .
git commit -m "initial import"
git push
```

Subsequent updates are handled by the workflow.
